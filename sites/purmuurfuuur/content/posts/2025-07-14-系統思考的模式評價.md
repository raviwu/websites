> 系統思考是一種思考方式，它強調理解事物之間的相互關聯和整體性，而非孤立地看待個別部分。 它幫助我們從更廣闊的視野來看待問題，並找出問題的根本原因和潛在的解決方案。 系統思考不僅僅是一種思考方法，更是一種看待世界和理解複雜問題的框架。

系統的定義系統是由相互關聯的元素組成的整體，這些元素為了共同目的而協作，具有超越各部分總和的功能。

![](../images/markus-spiske-IiEFmIXZWSw-unsplash.jpg)

#### 核心組成要素

存量（Stock）：系統中某個時間點的資源、能量或信息的累積量。特性是可以測量，變化相對緩慢，例如浴缸裡的水、銀行帳戶餘額、人的知識水平

流量（Flow）：在一段時間內改變存量的速率，分為流入（增加存量）和流出（減少存量）。例如水龍頭的水流、收入支出、學習遺忘的速度

回饋迴路（Feedback Loop）：迴路有兩種，增強迴路（Reinforcing Loop, R）產生指數成長或衰退，正向的像是「成功→信心→更多努力→更大成功」，負向的像是「失敗→沮喪→更少努力→更大失敗」。另一種是平衡迴路（Balancing Loop, B），尋求目標或平衡的自我修正行為，像是恆溫器、市場供需平衡等等

#### 系統會有一些行為模式

延遲（Delay）是原因和結果之間的時間差，常導致過度反應或政策阻力。例如運動效果的延遲、教育投資的延遲回報等。

非線性關係是指小改變可能產生大影響（槓桿點），大努力可能產生小效果

#### 系統結構概念

槓桿點（Leverage Point）是指系統中能用最小力量產生最大改變的地方。常見的高槓桿點像是改變心智模式、系統目的等；低槓桿點則是類似調整參數、改變個別規則

系統目的（Purpose）說的是系統的最佳功能指標是它實際產生的行為，不是宣稱的目的，而是實際展現的模式

心智模式（Mental Model）影響系統結構的深層信念和假設，往往是最難改變但影響最深遠的層面

#### 系統思考的關鍵原則

- 整體觀：關注關係和模式，而非事件
- 結構決定行為：問題的根源在系統結構，不是個人
- 症狀vs根本原因：治標不治本只會讓問題再現
- 短期vs長期：很多解決方案短期有效，長期有害

### 關係的系統狀態

#### 核心存量現況

雙方信任存量水位低落，持續快速流出，流入幾乎為零。到了臨界點。

執行任務不掉球的期待存量，水位在多年失望累積後過度透支（負值）。需要「系統性穩定」，而非單次表現才能補充。

自己的情緒調節存量目前水位極不穩定，經常歸零。高波動性，容易被外在事件瞬間耗盡。

#### 核心問題的系統映射

**問題1：任務執行的品質動盪**

系統結構診斷

> 意圖（想做好）→ 執行（品質不一）→ 結果（對方失望）→  自己焦慮上升 → 下次執行更不穩定

根本原因

- 注意力資源管理失效：把注意力當作無限資源
- 品質控制機制缺失：沒有身體內建的「檢查-確認」步驟
- 情緒干擾執行：焦慮狀態下執行品質下降

**問題2：關心表達的系統性偏差**

目前模式搭配度幾乎錯位

- 關心輸出：60%情感表達 + 30%補償行為 + 10%實際需求滿足
- 需求接收：10%情感 + 20%補償 + 70%實際持續支持

細微度缺失的系統原因

- 感知系統校準錯誤：關心雷達調錯頻率
- 回饋機制失效：沒有建立「關心效果」的測量系統

#### 負面增強迴路分析

**增強迴路R1：執行-信任螺旋下墜**

任務品質不穩定 → 預期降低 → 提醒變嚴厲 → 感受壓力 → 執行時更焦慮 → 品質更不穩定 →
信任進一步流失 → 更嚴厲...

系統特性：自我加速的惡性循環已進入高速運轉階段

**增強迴路R2：關心-失望循環**

用自己的方式關心 → 感受不到 → 表達失望 → 更努力用同樣方式關心 → 效果依然差 → 更失望 → 開始懷疑自己 → 關心變得更不自然...

系統特性：越努力，偏差越大，已開始質疑自己的關心能力

**增強迴路R3：溝通品質惡化**

對方疲憊 → 溝通更直接嚴厲 → 情緒化反應 →
溝通品質下降 → 問題解決效果差 → 更疲憊 →
下次溝通更嚴厲...

系統特性：溝通成本持續上升，每次對話都是高風險互動

#### 系統延遲效應的殘酷現實

**改善努力延遲鏈**

- 決定改變 (即時)
- 行為實際改變 (1-2週)
- 感受到改變 (1個月)
- 開始相信改變 (3-6個月)
- 關係穩定改善 (6個月-2年)

關鍵：目前處於「決定改變」階段，但期待「關係立即改善」，這個不實際的期待本身就會破壞系統。

**信任重建延遲**

- 觀察期：需要至少3-6個月持續觀察穩定性
- 測試期：會無意識地「測試」改變是否真實
- 接受期：即使看到改變，情感接受也需要時間

#### 系統槓桿點重新定位

最高槓桿點：將注意力從「事件驅動」調整成「系統驅動」

- 事件驅動：事件發生 → 注意力集中 → 處理 → 注意力分散 → 下個事件
- 系統驅動：系統檢查 → 注意力分配 → 執行 → 品質確認 → 系統更新

次高槓桿點：關心的方式不能光用「自己的方式」，要建立「對方視角」

- 我的方式：情感表達、陪伴、補償行為
- 對方視角：實際支持、系統穩定、被理解

#### 系統崩潰風險評估

關係終結的想法：✓ 已觸發

停止期待改變：接近觸發

情緒麻木或暴躁：風險中

#### 系統修復的最後窗口

時間：大約3-6個月

修復條件：

- 立即停止所有負面增強迴路
- 建立可測量的穩定性指標
- 看到「系統性改變」而非「努力表演」

#### 現實評估

- 存量：嚴重透支狀態
- 流量：負向流量遠大於正向流量
- 穩定性：高度不穩定，外在小刺激就可能引發大波動
- 修復能力：系統自癒能力幾乎耗盡

目前是在「搶救關係」，不是追求完美的時候，而是防止系統性崩潰的緊急狀態。改變必須可測量、持續、系統性，且不依賴對方來維持動力。在搶救階段，對方沒有多餘的能量給你正向能量，必須建立「自我維持」的改變系統。
